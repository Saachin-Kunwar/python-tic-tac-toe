## 🚀 Features

- 🎯 Player vs Player mode  
- 🤖 Player vs Computer mode  
- 🧠 Automatic win & draw detection  
- 🎨 Clean dark-themed UI  
- 📊 Live score tracking  
- 🔁 Restart / Play Again system  
- 🧾 Last result display  
- 📂 Menu bar integration  

---

## 🛠️ Tech Stack

- **Language:** Python 🐍  
- **GUI Library:** Tkinter  
- **Concepts Used:**
  - Object-Oriented Programming (OOP)
  - Event-driven programming
  - GUI state management

📌 Project Details
<details> <summary>🎮 How to Play</summary>
- Player X starts first  <br>
- Players take turns placing marks <br>
- First to align 3 marks (row, column, diagonal) wins <br>
- If all cells are filled → it's a Draw <br>
</details>

<details> <summary>📈 What I Learned</summary>
- Structuring projects using OOP (Class-based design)  <br>
- Managing state in GUI applications  <br>
- Handling user interactions and callbacks  <br>
- Designing clean and interactive UI  <br>
- Debugging real-world logic issues  <br>
</details>


<details> <summary>⚠️ Challenges Faced</summary>
- Managing multiple game states (win, draw, turn switching)  <br>
- Avoiding bugs caused by global variables  <br>
- Synchronizing UI updates with game logic  <br>
- Maintaining clean structure as project grows  <br>
</details>

<details> <summary>💡 Solutions</summary>
- Refactored into class-based architecture <br>
- Removed global variables  <br>
- Separated logic into reusable functions  <br>
- Improved UI feedback for better debugging  <br>
</details>
